개발자 그룹 커리큘럼(2주, 8시간)

음악을 들을 줄은 알지만 읽을 줄은 몰랐던 분들 대상입니다. 악보를 읽기 위해 음악의 기본적인 문법 체계를 배우고, 최종적으로는 약 8마디~16마디 정도의 아주 짧은 멜로디를 스스로! 작곡해봅니다.

1주차: 음악의 기본 문법(Syntax)

  1. 1.1 음악가들의 문법 프로그래밍 언어마다 나름의 문법 체계가 있듯, 음악에도 문법 체계가 있다. 언어의 기본은 문법! 음악의 문법부터 배워보자.
  2. 1.2 문법의 구성 요소들 프로그램을 구성하는 소스코드가 있듯, 음악을 구성하는 세부적인 요소도 여러가지가 있다. 음표, 박자, 리듬... 아주 모호하게만 알고있던 이 개념들을 정확히 알아보자.
  3. 1.3 문법 실습 손코딩 비슷하게, 직접 나눠주는 오선보에 아주 간단한 악보를 그려보며 앞서 배운 내용들을 하나의 작은 악보로 구현해본다.

2주차: 조성(Tonality)과 화음(Chord)

  1. 2.1 조성 다채로운 음악을 위한 기본 중 기본. 왜 어떤 음악은 뭔가 암울하고, 어떤 음악은 동양적으로 느껴지는지 파헤쳐보자.
  2. 2.2 조표 앞에서 배운 조성을 악보에 실제로 기보하는 법을 배운다. 조표(Key Signature)를 활용한다.
  3. 2.3 화음 듣기 좋은 음은 모두 화음인걸까? 멜로디가 두 개면 다 화음일까? 지금까지 애매하게 알고있던 화음 개념 바로 잡기.
  4. 2.4 화음 실습 8~16마디 짜리 짧은 악보를 지금까지 배운 문법, 조성 개념(조표), 화음을 활용해서 직접 그려본다. 그린 결과물을 모임장이 직접 시연해주며, 에러가 있는 경우 잡아준다.

음악가 그룹 커리큘럼(4주, 12시간)

컴퓨터가 마냥 무섭고, 소프트웨어 잘못 만지면 포맷될 것 같고.. 막연하게 IT에 대한 두려움만 가득한 음악가들 대상입니다. 웹 프로그래밍의 기초를 배우며 새로운 분야에 대한 불안감을 해소하고, 나의 연주회 초대장을 웹에서 구현해봅니다.

1주차: 나와 프로그래밍의 관계 재정리

  1. 사실 당신은 늘 프로그래밍과 함께 살고 있다 아침에 일어나자마자 알람 앱을 끄고, 모르는게 있으면 검색해서 결과를 보는 세상. 사실 프로그래밍은 나와 무척 가까은 분야인데, 느끼기 어려웠다. 첫 만남은 그동안 몰랐던 프로그래밍의 일면을 들여다보며 워밍업.
  2. 나의 첫 코드 작성 웹 페이지를 구성하는 기술 중 하나인 HTML. 우리가 아무 생각 없이 웹 서핑을 하면서 매일 만나는 기술이지만 몰랐다! 간단히 HTML을 알아보고, 간단한 수준의 코드도 작성해본다.

2주차: 무섭지 않은 HTML, CSS

  1. HTML과 좀 더 친해지기 내가 작성한 코드가 브라우저에서 바로 보이다니, 신기해 미칠 지경일테지만 알아야 하는 것이 좀 더 있다. 꼭 지켜줘야할 규칙, 더 편하게 코드를 작성하기 위해 알아야 할 개념들을 살펴보자.
  2. HTML의 단짝친구 CSS CSS 라는 기술을 새로 배워 웹 페이지에 스타일을 입혀본다. 다소 못생겨보이는(?) HTML 문서에 다채로운 변화를 주며 나의 자아를 표현해보자.

3주차: 까칠하지만 좋은 친구 JavaScript

  1. 프로그래밍 기초 개념 HTML, CSS를 활용해 만들어둔 나의 자아 충만한 웹 페이지를 조금 더 발전시켜주자. 자바스크립트(JavaScript) 라는 기술을 새로 배우며 변수, 데이터 타입, 연산자 등(워워~ 안 무서워요) 기초 개념을 알아본다.
  2. 내 웹 페이지에 작은 기능 추가하기 앞서 배운 기초 개념을 바로 활용하여, 조용했던 나의 웹 페이지에 인사말 출력 기능을 넣어준다.

4주차: 나의 연주회 초대장 구현 프로젝트

  1. 나의 연주회 초대장 만들기(1) 주어진 HTML, CSS, 그리고 JS 템플릿을 활용해 내 연주회 초대장을 웹 페이지로 만들어본다. 템플릿 코드를 수정하고, 필요한건 추가하는 과정에서 코드 작성에 조금 더 익숙해진다.
  2. 나의 연주회 초대장 만들기(2) 열심히 만든 페이지를 남에게 공유하려면? 어딘가 올라가서 남들에게 주소를 줄 수 있어야 할텐데.. 이건 어떻게 할 수 있을까? 함께 알아보며, 프로젝트 완성하기.
